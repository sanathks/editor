# Editor LLM Integration Guide

This file explains the LLM-safe way to create shareable Editor documents.

## Product
- App URL: https://editor.sanath.dev
- Editor is client-side markdown.

## Recommended flow (no compression library needed)
Use the short-link API flow:
1. Send markdown to `POST /api/doc`
2. API returns a short id and URL
3. Share the returned short URL (`/d/:id`)

This avoids `lz-string` and works out of the box for LLMs.

## API contract

### Create document
`POST /api/doc`

Request JSON:
```json
{
  "markdown": "# Title\n\nHello"
}
```

Response JSON (example):
```json
{
  "id": "abc123",
  "shortUrl": "https://editor.sanath.dev/d/abc123",
  "url": "https://editor.sanath.dev/d/abc123"
}
```

### Resolve document
`GET /d/:id`

- Returns the document route for that id (shareable link behavior)

## JavaScript example
```js
async function createEditorShareUrl(markdown) {
  const res = await fetch("https://editor.sanath.dev/api/doc", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ markdown })
  });

  if (!res.ok) throw new Error(`Create doc failed: ${res.status}`);

  const data = await res.json();
  return data.shortUrl || data.url;
}

// usage
const url = await createEditorShareUrl("# Hello\n\n- one\n- two");
console.log(url);
```

## curl example
```bash
curl -s -X POST "https://editor.sanath.dev/api/doc" \
  -H "Content-Type: application/json" \
  -d '{"markdown":"# Hello\n\n- one\n- two"}'
```

## Rules for LLM-generated content
1. Always produce valid markdown.
2. Send full markdown as one string in `markdown`.
3. Use `POST /api/doc` and return the `shortUrl`.
4. Do not include secrets in document content.

## Backward compatibility note
Legacy links using `#doc=` may still exist, but new automations should use `/api/doc` + `/d/:id`.
