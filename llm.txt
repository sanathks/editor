# Editor LLM Integration Guide

This file explains how to generate shareable Editor documents.

## Product
- App URL: https://editor.sanath.dev
- Editor is client-side markdown with URL-hash state.

## Share URL format
Editor stores markdown in the URL hash like this:

`https://editor.sanath.dev/#doc=<ENCODED_MARKDOWN>`

Where:
- `<ENCODED_MARKDOWN>` is produced by `LZString.compressToEncodedURIComponent(markdown)`
- Markdown is the full document text joined with `\n`

## Encoding algorithm
Use [lz-string](https://www.npmjs.com/package/lz-string):
- Encode: `compressToEncodedURIComponent(markdown)`
- Decode: `decompressFromEncodedURIComponent(encoded)`

## JavaScript example (generate share URL)
```js
import LZString from "lz-string";

const markdown = `# Project Plan

## Goals
- Ship v1
- Validate UX

| Task | Owner |
| --- | --- |
| Build editor | Rex |
| Review | Sanath |
`;

const encoded = LZString.compressToEncodedURIComponent(markdown);
const shareUrl = `https://editor.sanath.dev/#doc=${encoded}`;
console.log(shareUrl);
```

## JavaScript example (decode URL)
```js
import LZString from "lz-string";

const hash = "#doc=...";
const encoded = hash.startsWith("#doc=") ? hash.slice(5) : "";
const markdown = LZString.decompressFromEncodedURIComponent(encoded) || "";
console.log(markdown);
```

## Rules for LLM-generated content
1. Always output valid markdown.
2. Put the full document in one markdown string.
3. Encode with `compressToEncodedURIComponent`.
4. Return full URL with `#doc=`.
5. Do not include secrets in generated documents.

## Notes
- If markdown is empty, Editor uses URL without `#doc`.
- Very large documents can create long URLs (browser/share limits).
- Multiline table cells are supported in UI editing, but markdown serialization remains line-based.

## Minimal helper (copy/paste)
```js
function makeEditorUrl(markdown, base = "https://editor.sanath.dev") {
  const encoded = LZString.compressToEncodedURIComponent(markdown || "");
  return encoded ? `${base}/#doc=${encoded}` : `${base}/`;
}
```
